## object 和 map 的区别

-   键的类型：Map 的键可以是任意类型的值，包括对象、函数、基本数据类型等。而 Object 的键只能是字符串或者符号（Symbol）类型。

-   键值对的顺序：Map 保持插入顺序，即键值对的顺序与插入顺序一致。而 Object 没有保证属性的顺序，属性的顺序可能会发生变化。

-   迭代：Map 提供了内置的迭代器（Iterator）接口，可以使用 for...of 循环或者迭代器的 next()方法进行遍历。而 Object 的属性遍历需要通过 Object.keys()、Object.values()或者 Object.entries()等方法来实现。

-   大小的获取：Map 有一个 size 属性可以获取键值对的数量，而 Object 没有直接获取属性数量的方法，需要通过 Object.keys()等方法获取属性数组的长度。

-   继承：Object 是所有对象的基类，它有一些内置的方法和属性（如 toString()、hasOwnProperty()等），而 Map 是一个独立的数据结构，它没有继承自其他类。

-   引用类型作为键：在 Map 中，引用类型的键是以引用的方式进行比较的，即只有引用相同才会被认为是同一个键。而在 Object 中，键会被转换为字符串，因此不同的引用类型会被转换为相同的字符串键。

**总的来说，Map 适用于需要保持插入顺序并且键可以是任意类型的情况，而 Object 适用于简单的键值对存储，并且对属性的顺序没有特殊要求的情况。根据具体的需求和使用场景，选择合适的数据结构会更加有效和方便**
