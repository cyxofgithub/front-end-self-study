## 是什么

工厂模式是一种软件设计模式，**它提供了一种创建对象的方式，而无需直接使用构造函数或实例化具体的类。工厂模式将对象的创建和使用分离开来，通过一个工厂方法或工厂类来封装对象的创建过程**。

## 作用

工厂模式的主要目的是隐藏对象的创建细节，提供一种更灵活和可扩展的方式来创建对象。它可以使客户端代码与具体的类解耦，只依赖于抽象的接口或基类。这样，当需要更换具体的实现类时，客户端代码无需做任何修改，只需修改工厂方法或工厂类的实现即可。

客户端代码是指在软件系统中运行在客户端（通常是用户的计算机或设备）上的代码。它负责处理用户界面、用户输入、与服务器通信以及其他与用户交互相关的功能。

## 实现

```javascript
// 创建一个工厂函数作为工厂方法
function createCar(type) {
    let car;

    // 根据不同的类型创建不同的车辆对象
    if (type === "SUV") {
        car = new SUV();
    } else if (type === "Sedan") {
        car = new Sedan();
    } else if (type === "Hatchback") {
        car = new Hatchback();
    }

    // 设置公共属性和方法
    car.color = "black";
    car.start = function() {
        console.log("The car is starting.");
    };

    return car;
}

// 定义不同类型的车辆类
function SUV() {
    this.type = "SUV";
}

function Sedan() {
    this.type = "Sedan";
}

function Hatchback() {
    this.type = "Hatchback";
}

// 使用工厂方法创建不同类型的车辆对象
const suvCar = createCar("SUV");
console.log(suvCar); // 输出: SUV { type: 'SUV', color: 'black', start: [Function] }

const sedanCar = createCar("Sedan");
console.log(sedanCar); // 输出: Sedan { type: 'Sedan', color: 'black', start: [Function] }

const hatchbackCar = createCar("Hatchback");
console.log(hatchbackCar); // 输出: Hatchback { type: 'Hatchback', color: 'black', start: [Function] }
```

## 拓展

工厂模式有多种实现，如简单工厂模式，工厂方法模式，抽象工厂模式
