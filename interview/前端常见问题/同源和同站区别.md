-   同源：严格的 “协议 + 完整域名 + 端口” 一致，管 “资源访问权限”（如 AJAX、DOM 操作）；
-   同站：宽松的 “注册域名” 一致，管 “Cookie 跨站携带”（如 SameSite 策略）；

以基准 URL https://www.example.com:8080 为例，对比不同 URL 与它的 “同源”“同站” 关系：
目标 URL 与基准的关系 判断逻辑

-   https://www.example.com 同源、同站 协议（https）、域名（www.example.com）一致，端口默认（443 vs 8080 不影响同站），满足同源 + 同站
-   http://www.example.com:8080 不同源、同站 协议不同（http vs https）→ 不同源；注册域名（example.com）一致 → 同站
-   https://blog.example.com 不同源、同站 域名不同（blog vs www）→ 不同源；注册域名（example.com）一致 → 同站
-   https://www.example.co.uk 不同源、不同站 域名不同（example.co.uk vs example.com）→ 不同源；注册域名不同 → 不同站
-   https://www.another.com 不同源、不同站 域名不同（another.com vs example.com）→ 不同源；注册域名不同 → 不同站
-   https://user.github.io 不同源、不同站 注册域名（user.github.io）与基准（example.com）不同 → 不同站、不同源
