函数式编程（Functional Programming, FP）和面向对象编程（Object-Oriented Programming, OOP）是两种常见的编程范式，它们在编程思想、组织方式、数据管理与副作用处理等方面有显著区别：

---

## 一、核心思想

### 函数式编程

-   **核心理念**：将计算视为函数的组合，强调数据不可变和无副作用，侧重“做什么”而不是“怎么做”。
-   **函数是一等公民**：函数可以赋值给变量、作为参数或返回值传递。
-   **追求纯函数**：输入和输出严格对应，无副作用（不依赖外部、也不改变外部状态）。
-   **不可变数据**：数据定义后不可更改，如需变化需生成新数据。
-   **强调声明式**：用表达式和组合的方式描述“是什么”，而非命令式地“怎么做”。

### 面向对象编程

-   **核心理念**：将现实事物抽象成对象，通过封装、继承、多态等特性组织代码，侧重“怎么做”。
-   **对象**：数据和行为（属性与方法）的组合体。
-   **封装**：内部实现隐藏，对外只暴露接口。
-   **继承**：子类继承父类（属性、方法）。
-   **多态**：统一调用接口，不同对象有不同实现。
-   **可变状态**：对象的状态可随时被修改。

---

## 二、主要区别对比

|          | 函数式编程                       | 面向对象编程                       |
| -------- | -------------------------------- | ---------------------------------- |
| 组成单元 | 函数                             | 对象                               |
| 数据管理 | 不可变，数据只读，变化生成新数据 | 可变，状态保存在对象内部           |
| 副作用   | 不鼓励副作用，尽量纯函数         | 可以有副作用，如方法内修改成员变量 |
| 抽象方式 | 用高阶函数、函数组合抽象操作     | 用类、对象、继承实现抽象           |
| 复用手段 | 函数组合、高阶函数               | 继承、接口、多态、组合             |
| 并发安全 | 更易实现（无共享状态）           | 要注意并发时的状态一致性           |
| 代码风格 | 声明式、表达式驱动               | 命令式、语句驱动                   |

如何理解并发安全？

并发调用同一个函数时可能改变对象的状态，而函数不会

## 三、简单示例

### 1. 函数式编程示例（JavaScript）

```js
// 纯函数：无副作用、数据不可变
const nums = [1, 2, 3, 4];
const doubled = nums.map((x) => x * 2); // 不改变 nums，生成新数组
```

### 2. 面向对象编程示例（JavaScript）

```js
class Counter {
    constructor() {
        this.count = 0;
    }
    increment() {
        this.count += 1; // 改变对象内部状态
    }
}
const c = new Counter();
c.increment(); // count 变成 1
```

## 四、应用场景举例

-   **函数式编程**更适合数据处理、不可变数据流、分布式和并发安全场景，例如前端 React（函数式组件）、Haskell、Scala。
-   **面向对象编程**适合描述具备丰富状态和行为的现实世界结构，如大型企业级系统、游戏开发等，如 Java、C++、Python。

---

## 五、小结

-   **FP**：高度抽象、易于组合测试、适合不可变数据流和并发，代码简洁但对初学者有门槛。
-   **OOP**：结构清晰，适合建模现实世界、复杂系统，维护性好但易出现过度继承和耦合问题。

两者不是对立——实际开发常常混合：如在 OOP 语言中用函数式风格，在 FP 中也用对象（如数据结构）。关键是根据具体场景选择合适的范式与风格。
